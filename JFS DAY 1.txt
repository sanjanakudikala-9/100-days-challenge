<?> any type 
<? extends T> >> (T or any child datatype of T)
any type related to T or datatype that is under inheritance of T(below the inheritance tree of T)
this only supports reading >> we can't add because the compiler don't the exact datatype
<? super T> >> T datatype or any parent of T 
this supports write because the compiler know it exact datatype because inheritance is in upward direction 
so we can add elements but reading is limited

synchronization >> helps in avoiding the conflicts in mutlithreading as multiple threads share same memory 
3 ways of synchronization 
1)synchronized block :Synchronized blocks allow locking only a specific section of code instead of the entire method. This makes the program more efficient by reducing the scope of synchronization.
2)synchronized method : used to lock an entire method so that only one thread can execute it at a time for a particular object
but reduce performance 
3) static synchronization :used when static data or methods need to be protected in a multithreaded environment.

Locks are advanced > instead of synchronization we use locks
