DATA AUTOMATION - INTERNSHIP - PROJECT :

Automated Student Performance Analytics System
During my 15-day internship at MSME, I worked on a Data Automation project focused on 
educational institute analytics. The manual process of tracking student performance across 
terms via Excel was highly inefficient. I was tasked with building an automated pipeline: 
setting up a relational MySQL database to store the data securely, and then connecting it 
to Power BI using DAX to automatically calculate core KPIs like pass/fail percentages, 
subject averages, and top-performing students. This eliminated manual spreadsheet calculations entirely.

What was the biggest challenge you faced during this automation?

Answer: "Handling missing or mismatched data types during the initial data import from CSV 
to MySQL. I had to clean the data types and ensure that StudentID matched perfectly across
 both tables to maintain referential integrity."



in XAMPP : 
USE StudentAnalytics;

-- Create the master table for student profiles
CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    FirstName VARCHAR(50),
    LastName VARCHAR(50),
    Gender VARCHAR(10),
    GradeLevel VARCHAR(10)
) ENGINE=InnoDB;

ENGINE=InnoDB >>> ensures the relationship between the tables 


-- Create the table for tracking marks
CREATE TABLE Marks (
    MarkID INT AUTO_INCREMENT PRIMARY KEY,
    StudentID INT,
    Subject VARCHAR(50),
    MarksObtained INT,
    MaxMarks INT DEFAULT 100,
    ExamTerm VARCHAR(20),
    FOREIGN KEY (StudentID) REFERENCES Students(StudentID)
) ENGINE=InnoDB;
